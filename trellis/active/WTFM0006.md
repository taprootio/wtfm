# WTFM0006 - Help-document output mode conforming to ESP0119 strip-to-bare

wtfm's reference output is demo-first: descriptions render into interactive
`<wtfm-code-block>` elements carrying base64-encoded source and embedded JSON
scripts, wrapped in styled `doc-header`/`doc-section` divs. Espalier's
URL-help renderer (**ESP0119**) strips fetched help HTML to bare semantic
elements — that output would be gutted. This task adds the **help-document
mode**: prose-first output shaped to be fetched and injected into an
`esp-flyout`.

## Scope

- **A second render path.** Add an exported `renderHelpDocument()` function and
  an Eleventy `renderHelpDocs(surfaceSlug, markdown)` shortcode. Help Markdown
  is authored separately from CEM reference descriptions and coexists with
  `renderSurfaceDocs` in the same build.
- **Markup contract.** Render with raw HTML and MathJax disabled. Allow only
  semantic headings, paragraphs and inline emphasis, lists, tables, fenced
  code, blockquotes, images, links, and simple separators/line breaks. Allow
  only anchor `id` plus link/image `href`/`src`/`alt`; emit no wrappers,
  classes, styles, scripts, framework attributes, or `<wtfm-code-block>`.
- **Anchors per field.** Authors pin exact, case-sensitive field ids with
  WTFM0004's `## Title {#Title}` syntax. Duplicate ids fail rendering. One
  document serves a whole surface, so per-field help retains surrounding
  context.
- **Link/image URL shape.** Resolve document-relative `href`/`src` values
  against the surface's help route and turn them into root-absolute URLs;
  WTFM0008's Eleventy HTML-base pass then applies `/help/`. Leave external,
  protocol, fragment-only, and data URLs untouched as appropriate.
- Tests: a fixture surface rendered in help mode, asserted against the exact
  tag/attribute keep-list, URL resolution rules, and expected anchor ids.

## Notes

- ESP0119 is still active and has no released renderer to test today. Pin this
  task to its written keep-list and move released-renderer compatibility to
  WTFM0003's final external integration gate.
- Help documents are trusted, app-owned content — ESP0119 normalizes but does
  not sanitize; wtfm's job is shape, not security.
- Consumed by Taproot TR00385 (authoring help content) and TR00386 (editor
  adoption).

## Risks

- Keep-list mismatch: too rich and the flyout strips meaning away, too lean
  and tables/code lose fidelity — test against the actual normalization
  early, not after authoring a full content set.
- Dual-output complexity: reference and help modes share renderers where
  possible; avoid forking section logic into two divergent code paths.
