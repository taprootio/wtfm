# WTFM0003 - Epic — wtfm support for Taproot's Help & Docs Surfaces initiative

**This is the tracking/epic item for the `Help & Docs Surfaces` milestone.** It
owns sequencing and the cross-repo contracts; implementation lives in the child
tasks. It is the wtfm-side counterpart of Taproot's **TR00382** epic.

## The goal

Taproot documents its UI as **one page per surface** (a surface being a unit of
display — the page editor, each page-type form), generated by wtfm from
Taproot's Custom Elements Manifest, served under `/help/` on the app origin,
and consumed two ways: read directly as docs, and fetched by Espalier's
URL-based help system (**ESP0119**) into an `esp-flyout` beside the editor via
`help-src#<field-name>` anchors.

## Cross-repo map

- **Taproot (`../taproot`, `TR` backlog):** TR00382 (epic), TR00383 (CEM
  pipeline), TR00384 (docs/help site + `/help/` serving), TR00385 (annotate +
  author help docs), TR00386 (adopt ESP0119 in the editor), TR00387 (live
  preview; independent of wtfm).
- **Espalier (`taproot-controls` Trellis backlog):** ESP0119 (URL-help
  renderer, active), ESP0136 (preview panel, active) — consumed as released,
  no wtfm coupling beyond the ESP0119 markup/anchor contract.
- **wtfm (this backlog):** the child tasks below. Cross-repo ids appear in
  prose only — `depends_on` accepts WTFM ids alone.

## Shared contracts (decide once, here)

- **Anchor convention:** heading `id`s are deterministic, documented, and
  overridable so a help document's section anchors can exactly match Espalier
  `field-name`s (`help-src#<field-name>`). Owned by WTFM0004; consumed by
  WTFM0006/WTFM0007 and Taproot TR00385/TR00386.
- **ESP0119 markup contract:** help-document output survives strip-to-bare
  normalization — semantic elements only (headings, prose, lists, tables,
  code, blockquotes, images, links) plus anchor `id`s and `href`/`src`/`alt`.
  No `<wtfm-code-block>`, no class-dependent styling. Owned by WTFM0006.
- **Surface model:** a surface is declared in component JSDoc, has a stable
  slug (its URL becomes a permanent help link target), and lists its member
  components in display order. Owned by WTFM0005.
- **URL shape:** all generated URLs, breadcrumbs, and asset references compose
  with an Eleventy `pathPrefix` (Taproot serves at `/help/`). Owned by
  WTFM0008.

## Notes

- **Tracking rule:** wtfm-repo work is tracked **here**, in wtfm's own
  backlog, since the repo's Trellis onboarding (2026-07-23). This supersedes
  TR00382's earlier "all wtfm work is tracked in the Taproot backlog" rule,
  which predated this backlog; TR00382 has been updated to match. Taproot
  tasks reference WTFM ids in prose.
- **Done when** every child task is complete and Taproot's TR00384/TR00385
  have successfully built surface pages and help documents against a released
  wtfm — the epic stays open until the capabilities have shipped *and* been
  consumed.
- Effort 2 is coordination only — the shared-contract decisions above. Total
  initiative effort is the sum of the children.

## Risks

- Scope creep: the initiative needs help docs for Taproot's **authoring
  surfaces** first, not "document all 154 components." Hold the line
  (mirrors TR00382's risk).
- Contract drift against ESP0119: implement WTFM0006 against the ESP0119
  contract as **released**, not its backlog item's provisional API (mirrors
  TR00386's rule).
