# WTFM0003 - Epic — wtfm support for Taproot's Help & Docs Surfaces initiative

**This is the tracking/epic item for the `Help & Docs Surfaces` milestone.** It
owns sequencing and the cross-repo contracts; implementation lives in the child
tasks. It is the wtfm-side counterpart of Taproot's **TR00382** epic.

## The goal

Taproot documents its UI as **one page per surface** (a surface being a unit of
display — the page editor, each page-type form), generated by wtfm from
Taproot's Custom Elements Manifest, served under `/help/` on the app origin,
and consumed two ways: read directly as docs, and fetched by Espalier's
URL-based help system (**ESP0119**) into an `esp-flyout` beside the editor via
`help-src#<field-name>` anchors.

## Shared contracts (decide once, here)

- **Reference vs help content:** CEM component metadata composes the reference
  page for a surface. Field-level help is separately authored Markdown because
  Taproot's `esp-form-item[field-name]` values are internal template details,
  not CEM attributes. Both outputs share the surface slug and URL metadata.
- **Anchor convention:** generated ids are deterministic lowercase kebab-case;
  generated headings on a composed reference surface are namespaced by member.
  Explicit `{#ExactId}` and CEM `@helpAnchor ExactId` overrides preserve exact,
  case-sensitive ids for `help-src#<field-name>`. Any duplicate id fails the
  build rather than silently renumbering an existing link target. Owned by
  WTFM0004; consumed by WTFM0005/WTFM0006/WTFM0007 and Taproot TR00385/TR00386.
- **ESP0119 markup contract:** help-document output survives strip-to-bare
  normalization — semantic elements only (headings, prose, lists, tables,
  code, blockquotes, images, links) plus anchor `id`s and `href`/`src`/`alt`.
  No `<wtfm-code-block>`, no class-dependent styling. Owned by WTFM0006.
- **Surface model:** one defining component carries `@docSurface <slug>`,
  `@docSurfaceTitle <title>`, and `@docSurfaceParts <ordered, comma-separated
  custom-element tags>`. Existing menu tags supply nav metadata. A component
  can be listed by multiple surfaces; unknown/ambiguous members and duplicate
  surface slugs are build errors. Reference and help routes default to
  `/surfaces/<slug>/` and `/surfaces/<slug>/help/`. Owned by WTFM0005.
- **URL shape:** all generated URLs, breadcrumbs, and asset references compose
  with Eleventy's `HtmlBasePlugin` and `pathPrefix` (Taproot serves at
  `/help/`). Internal URLs are root-absolute; help-document-relative URLs are
  first resolved against the help page. Owned by WTFM0008.
- **Manifest contract:** `help-manifest.json` is versioned and contains each
  surface's slug, final reference/help URLs, and ordered help-heading ids. A
  consumer CLI fails for missing expected anchors; extras warn unless strict
  mode is requested. Owned by WTFM0007.

## Notes

- **Tracking rule:** wtfm-repo work is tracked **here**, in wtfm's own
  backlog, since the repo's Trellis onboarding (2026-07-23). This supersedes
  TR00382's earlier "all wtfm work is tracked in the Taproot backlog" rule,
  which predated this backlog; TR00382 has been updated to match. Taproot
  tasks reference WTFM ids in prose.
- **Implementation-ready when** every child task is complete, the package build
  and tests pass, and a real Eleventy fixture verifies the combined `/help/`
  surface/reference/help/manifest flow.
- **Done when** the implementation-ready package has been released, compatibility
  has been checked against the released ESP0119 renderer, and Taproot's
  TR00384/TR00385 have successfully consumed the release. The child tasks may
  close before those external events; this epic stays open until the
  capabilities have shipped *and* been consumed.
- Effort 2 is coordination only — the shared-contract decisions above. Total
  initiative effort is the sum of the children.

## Risks

- Scope creep: the initiative needs help docs for Taproot's **authoring
  surfaces** first, not "document all 154 components." Hold the line
  (mirrors TR00382's risk).
- Contract drift against ESP0119: implement WTFM0006 against the ESP0119
  contract as **released**, not its backlog item's provisional API (mirrors
  TR00386's rule).
