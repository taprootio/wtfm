# WTFM0001 - Publish @taprootio/wtfm to the public npm registry

**Priority:** High
**Effort:** 3
**Milestone:** Alpha
**Owner:** jereme

WTFM currently publishes to GitHub Packages (`npm.pkg.github.com`) as a private
package, which forces every consumer — including CI in this repo and in
`taproot-controls` — to carry GitHub auth tokens just to install it. Publish to
the public npm registry as `@taprootio/wtfm` instead (the `taprootio` npm org
already exists and hosts `@taprootio/trellis`).

The lockfile contains zero `npm.pkg.github.com` references — nothing is
*installed* from GitHub Packages — so the registry plumbing here is
publish-side and auth-side only.

## Scope

- `package.json`: change `publishConfig` to
  `{ "access": "public", "registry": "https://registry.npmjs.org/" }`.
- Delete `.npmrc` (its only line routes `@taprootio` to GitHub Packages; no
  installed dependency resolves there).
- `.github/workflows/main.yml` (publish job):
  - `setup-node` `registry-url` → `https://registry.npmjs.org`.
  - `npm publish` auth: `NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}` instead of
    `GITHUB_TOKEN`; drop the now-unneeded `packages: write` permission.
  - Consider `npm publish --provenance` (requires `id-token: write` and a
    public source repo — skip if the repo stays private).
  - Create and push the `v<version>` git tag only after `npm publish`
    succeeds (or base the should-publish check on the npm registry instead of
    git tags). Today the tag is pushed first, so a failed publish — likely on
    the first attempt against a new registry with a new token — leaves a tag
    that makes every retry skip publication.
- `.github/workflows/main.yml` + `ci.yml` (install steps): remove the
  `GITHUB_TOKEN: ${{ secrets.PKG_READ_TOKEN }}` env from `npm ci` — nothing in
  the lockfile needs it.
- `.github/workflows/backlog.yml`: once `.npmrc` is gone, drop the
  `--@taprootio:registry=…` override and run plain
  `npx --yes @taprootio/trellis check`.
- Repo settings (manual): create a granular npm automation token for the
  `@taprootio` scope and add it as the `NPM_TOKEN` actions secret.
- Downstream, in `../taproot-controls`: it pins `@taprootio/wtfm@^0.12.3` and
  routes the whole `@taprootio` scope to GitHub Packages via its `.npmrc`.
  After the first public publish, remove (or narrow) that scope routing, verify
  no other `@taprootio` package there still resolves from GitHub Packages,
  refresh `bun.lock`, and confirm the docs build.
- Decide what to do with the existing GitHub Packages versions: leave them
  readable until `taproot-controls` has migrated, then optionally deprecate.

## Acceptance criteria

- `npm view @taprootio/wtfm --registry=https://registry.npmjs.org` shows the
  released version.
- A fresh clone runs `npm ci && npm test` with no GitHub Packages auth
  configured.
- `taproot-controls` installs `@taprootio/wtfm` from the public registry and
  its docs build passes.
- `backlog.yml` runs `npx --yes @taprootio/trellis check` with no registry
  override.
- A failed `npm publish` leaves no `v<version>` tag behind, so re-running the
  workflow retries the publish instead of skipping it.
