# WTFM0004 - Emit stable heading anchors across rendered docs

wtfm output currently has **no anchors at all**: section renderers emit bare
`### Title` markdown (`build-doc-section.js`), and the plugin's markdown-it
instance has no anchor plugin, so no heading anywhere in a generated site
carries an `id`. Espalier's URL-help (`help-src#<field-name>`, ESP0119) and
plain deep-linking both need stable ids. This is the foundation task of the
`Help & Docs Surfaces` epic (WTFM0003).

## Scope

- Add one exported anchor module used by section renderers, composed surface
  rendering, and the plugin's Markdown library.
- Generated ids use a documented lowercase kebab-case convention. They are
  deterministic and stable across rebuilds.
- Add Markdown heading ids with `markdown-it-anchor` plus id-only explicit
  attributes (`## Title {#ExactId}`). Disable the plugin's extra `tabindex`
  attribute so generated headings gain only `id`.
- Emit explicit ids for every generated section heading (`Attributes`,
  `Slots`, …) and per-item heading. When composing a reference surface,
  namespace generated ids by the member's custom-element tag.
- Honor `@helpAnchor <ExactId>` on any CEM declaration/item passed to a
  renderer. Register it in the cem-plugin defaults and document the matching
  `jsDocTagsPlugin` entry.
- Preserve exact case for explicit ids. Never namespace an explicit help id.
  Duplicate generated or explicit ids in one rendered document fail the build;
  do not silently append order-dependent suffixes.
- Tests: slug determinism, namespace behavior, exact override precedence,
  collision failures, Markdown-authored ids, and representative rendered
  output.

## Notes

- ESP0119's strip-to-bare normalization explicitly preserves anchor `id`s —
  ids on semantic headings survive into the flyout.
- Anchors become load-bearing URLs the moment Taproot ships help links —
  treat any change to the slug convention after that as a breaking change.

## Risks

- Existing consumers (the taproot-controls docs site) must render unchanged
  apart from gaining ids — keep the change additive; no markup restructuring.
- Explicit ids are load-bearing and case-sensitive. The collision failure must
  name both the duplicated id and the surface/document being rendered.
