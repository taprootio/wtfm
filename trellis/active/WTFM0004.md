# WTFM0004 - Emit stable heading anchors across rendered docs

wtfm output currently has **no anchors at all**: section renderers emit bare
`### Title` markdown (`build-doc-section.js`), and the plugin's markdown-it
instance has no anchor plugin, so no heading anywhere in a generated site
carries an `id`. Espalier's URL-help (`help-src#<field-name>`, ESP0119) and
plain deep-linking both need stable ids. This is the foundation task of the
`Help & Docs Surfaces` epic (WTFM0003).

## Scope

- Emit `id` attributes on the headings produced by every section renderer —
  the section headings (`Attributes`, `Slots`, …) and each per-item heading
  (`build-doc-section.js`'s `### ${title}`) — via a single shared slug
  function.
- Add heading-anchor generation to the plugin's markdown-it setup
  (markdown-it-anchor or equivalent) so consumer-authored markdown/guide
  content gets ids from the same slug convention.
- Define and document the slug convention: deterministic, stable across
  rebuilds, predictable from the source name, with collision handling that
  never silently renumbers an existing anchor.
- Add an explicit override: a JSDoc tag (e.g. `@helpAnchor <id>`) honored by
  the renderers, so an anchor can be pinned to an Espalier `field-name`
  independent of the display title. Register the tag in the cem-plugin /
  documented `jsDocTagsPlugin` tag set.
- Tests: slug determinism, collision behavior, override precedence, and ids
  present in rendered output for a representative declaration.

## Notes

- ESP0119's strip-to-bare normalization explicitly preserves anchor `id`s —
  ids on semantic headings survive into the flyout.
- Anchors become load-bearing URLs the moment Taproot ships help links —
  treat any change to the slug convention after that as a breaking change.

## Risks

- Existing consumers (the taproot-controls docs site) must render unchanged
  apart from gaining ids — keep the change additive; no markup restructuring.
- Heading ids that collide across composed pages (multiple components on one
  surface page, WTFM0005) need deterministic disambiguation — coordinate the
  slug function with the surface-page composition.
