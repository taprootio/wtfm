# WTFM0002 - Switch npm publishing to Trusted Publishing (OIDC)

**Priority:** High
**Effort:** 2
**Milestone:** Alpha
**Owner:** jereme

WTFM0001 shipped the first public npm release authenticated by a granular
automation token stored as the `NPM_TOKEN` actions secret. That token was
created 2026-07-23 with a **7-day expiration (expires ~2026-07-30)** â€” after
that, publishes fail until it is rotated. npm's Trusted Publishing removes the
token dependency: npmjs.com is told to trust this repo's publish workflow, and
`npm publish` authenticates with a short-lived GitHub Actions OIDC token minted
per run. Nothing to store, rotate, or leak.

## Notes

- Trusted publishing requires a cloud-hosted runner (`ubuntu-latest` is fine)
  and the workflow filename to match exactly; if `main.yml` is ever renamed,
  the trusted-publisher config must be updated to match.
- Provenance attestations are only generated for public source repos; this
  repo is private, so OIDC publishes ship without provenance. If publish ever
  errors on provenance generation, publish with `--provenance=false`. Revisit
  if the repo goes public.
- Sequencing is safe in either order: with npm â‰Ą 11.5.1, OIDC is attempted
  when configured and token auth is the fallback, so the workflow change can
  merge before or after the npmjs.com configuration. Just land both before
  the token expires.
