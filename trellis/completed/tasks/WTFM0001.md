# WTFM0001 - Publish @taprootio/wtfm to the public npm registry

**Priority:** High
**Effort:** 3
**Milestone:** Alpha
**Owner:** jereme

WTFM currently publishes to GitHub Packages (`npm.pkg.github.com`) as a private
package, which forces every consumer — including CI in this repo and in
`taproot-controls` — to carry GitHub auth tokens just to install it. Publish to
the public npm registry as `@taprootio/wtfm` instead (the `taprootio` npm org
already exists and hosts `@taprootio/trellis`).

The lockfile contains zero `npm.pkg.github.com` references — nothing is
*installed* from GitHub Packages — so the registry plumbing here is
publish-side and auth-side only.

## Scope

- `package.json`: change `publishConfig` to
  `{ "access": "public", "registry": "https://registry.npmjs.org/" }`, and bump
  the version to `0.13.0` so merging this change triggers the first public
  publish (the `v0.12.3` tag already exists, so without a bump the publish job
  skips).
- Delete `.npmrc` (its only line routes `@taprootio` to GitHub Packages; no
  installed dependency resolves there).
- `.github/workflows/main.yml` (publish job):
  - `setup-node` `registry-url` → `https://registry.npmjs.org`.
  - `npm publish` auth: `NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}` instead of
    `GITHUB_TOKEN`; drop the now-unneeded `packages: write` permission.
  - Create and push the `v<version>` git tag only after `npm publish`
    succeeds, keeping the tag-based should-publish check. Today the tag is
    pushed first, so a failed publish — likely on the first attempt against a
    new registry with a new token — leaves a tag that makes every retry skip
    publication. (Accepted residual gap: if publish succeeds but the tag push
    fails, a re-run retries the publish of an existing version and gets a
    registry error; noise, not damage.)
- `.github/workflows/main.yml` + `ci.yml`: remove the
  `GITHUB_TOKEN: ${{ secrets.PKG_READ_TOKEN }}` env from every `npm ci` —
  nothing in the lockfile needs it — and drop the now-unneeded
  `packages: read` permission from the test/typecheck jobs.
- `.github/workflows/backlog.yml`: once `.npmrc` is gone, drop the
  `--@taprootio:registry=…` override and run plain
  `npx --yes @taprootio/trellis check`.
- Repo settings (manual, before merge): create a granular npm automation token
  for the `@taprootio` scope and add it as the `NPM_TOKEN` actions secret.
  Afterwards the `PKG_READ_TOKEN` secret is entirely unused and can be deleted.

## Notes

- `npm publish --provenance` is skipped: it requires a public source repo and
  this repo is private.
- The downstream migration of `taproot-controls` is **not** part of this task;
  it is tracked in that repo. Blocker discovered while scoping: registry
  routing is per-scope, and `taproot-controls`' `bun.lock` resolves
  `@taprootio/taproot-controls@1.140.5` (a published copy of itself, via
  wtfm's optional peer dependency) from GitHub Packages, where it must keep
  resolving until that repo either publishes publicly or stops installing
  itself. Its `.npmrc` scope routing therefore cannot simply be deleted.
- Existing GitHub Packages versions of `@taprootio/wtfm` stay readable until
  `taproot-controls` has migrated; optionally deprecate them afterwards.

## Acceptance criteria

- `npm view @taprootio/wtfm --registry=https://registry.npmjs.org` shows
  `0.13.0` after the merge to `main` publishes.
- A fresh clone runs `npm ci && npm test` with no GitHub Packages auth
  configured.
- No workflow references `PKG_READ_TOKEN` or GitHub Packages.
- `backlog.yml` runs `npx --yes @taprootio/trellis check` with no registry
  override.
- A failed `npm publish` leaves no `v<version>` tag behind, so re-running the
  workflow retries the publish instead of skipping it.
