# WTFM0005 - Surface pages — one doc page per unit of display

wtfm's model today is strictly **one page per component**: `renderDocs(name)`
renders a single CEM declaration, and `data/components.js` builds nav from
per-component `@docUrl` tags. Taproot's Help & Docs Surfaces initiative
(WTFM0003, Taproot TR00382) documents the app as **one page per surface** — a
unit of display such as the page editor or an individual page-type form —
where a surface typically comprises several components (the editor shell plus
its field components). The Espalier flyout links to surface pages.

## Scope

- **Declaration model.** One defining component carries `@docSurface <slug>`,
  `@docSurfaceTitle <title>`, and `@docSurfaceParts <ordered,
  comma-separated custom-element tags>`. Existing `@menuLabel`, `@menuGroup`,
  `@menuIcon`, and `@menuOrder` tags provide optional nav metadata. The parts
  list explicitly includes the defining component when desired. A component
  may appear in multiple lists or remain surface-less.
- Register the three new tags in the cem-plugin defaults and document matching
  `jsDocTagsPlugin` configuration.
- **Data helper.** A `data/surfaces.js` export (sibling of
  `data/components.js`): one pagination-ready entry per surface — slug,
  reference URL, help URL, page title, menu metadata, and ordered member
  declarations. Defaults are `/surfaces/<slug>/` and
  `/surfaces/<slug>/help/`; options may provide alternate route builders.
- **Rendering.** Add `renderSurfaceDocs(slug)` to compose member reference docs
  in declared order. Per-member rendering still honors `@docSections`, while
  WTFM0004 namespaces generated member/section/item anchors.
- **Validation.** Duplicate slugs, duplicate parts, unknown/ambiguous custom
  element tags, invalid slugs, or missing titles fail with actionable errors.
- **Stability.** Surface slugs/URLs are permanent link targets (help links
  point at them) — document that renames are breaking.
- Tests: surface collection from a fixture CEM, member ordering, and composed
  rendering.

## Notes

- Per-component pages keep working unchanged — surfaces are additive.
- Reference surfaces and authored help documents are separate outputs sharing
  this metadata. Form `field-name`s are internal template details and are not
  inferred from CEM attributes.
- Consumed by Taproot TR00384 (site + nav) and TR00385 (help docs per
  surface); the help-document variant of a surface page is WTFM0006.

## Risks

- Authoring-model bikeshed: keep the tag set minimal (slug, title, order) —
  richer grouping can layer on later without breaking the declaration
  contract.
- A surface spanning components from multiple modules relies on CEM analysis
  having captured them all — invalid membership fails the build rather than
  emitting a partial page.
